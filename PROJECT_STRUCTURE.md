# 项目结构说明（通俗版）

本文用通俗语言说明项目里每个文件/文件夹是干什么的，方便非技术人员大致了解。

---

## 整体是啥

这个项目是一个「音频转文字再翻译」的小工具：你上传一段音频，程序先把它转成文字，再按你选的语言翻译成另一种语言。  
项目分两块：**后端**（在服务器上干活、负责调用外部语音/翻译服务）、**前端**（你在浏览器里看到的页面和操作）。

---

## 根目录（项目最外层）

| 文件 | 通俗说明 |
|------|----------|
| **.gitignore** | 一份「不要上传到网上的清单」。里面列了像密码文件、临时文件、本机依赖包等，这样同步到 GitHub 时不会把这些东西传上去，更安全、也更干净。 |
| **README.md** | 项目的「使用说明书」：怎么在本地跑起来、怎么用 Docker 跑、需要配置哪些东西、接口和安全说明等。 |
| **docker-compose.yml** | 「一键启动配置」。告诉电脑：同时启动后端和前端两个程序，分别用哪个端口、怎么连在一起。适合想用 Docker 一键跑全站的人。 |

---

## backend/（后端：真正干活的程序）

后端负责：接收你上传的音频、调用语音转文字服务、再调用翻译服务，把结果通过接口还给前端。**钥匙（API Key）只放在后端，前端碰不到。**

### 后端根目录

| 文件 | 通俗说明 |
|------|----------|
| **.dockerignore** | 做后端「打包成镜像」时，列出哪些东西不要打进包里（比如本机的密钥、缓存）。这样包更小，也不会误把敏感信息打包进去。 |
| **.env.example** | 环境变量的「示例表格」。告诉你需要填哪些项（比如 API 密钥、允许访问的网址），你照抄一份改成 `.env` 并填上自己的值，程序就从 `.env` 里读，而 `.env` 不会上传到网上。 |
| **Dockerfile** | 后端的「打包说明书」：用啥基础环境、装哪些依赖、把哪部分代码放进去、最后用啥命令启动。Docker 根据它来建后端镜像。 |
| **requirements.txt** | 后端用到的所有 Python 依赖包名单（及版本）。装环境时按这个清单一次性装好，大家环境一致。 |

### backend/app/（后端主程序目录）

| 文件 | 通俗说明 |
|------|----------|
| **__init__.py** | 让 Python 把这一层目录认作一个「包」，方便别处引用。没有实际业务逻辑。 |
| **config.py** | 「配置中心」。从系统环境变量（或 .env）里读取：用哪个密钥、允许哪些网站访问、上传文件最大多大等，程序其他地方都来这里拿配置，不在代码里写死密钥。 |
| **main.py** | 后端的「入口」。把整个网站服务拉起来、挂上允许跨站访问的规则、把「转写」和「翻译」的网址路由接上，并提供一个简单的健康检查地址。 |

### backend/app/api/（对外提供的接口层）

| 文件 | 通俗说明 |
|------|----------|
| **__init__.py** | 同上，标记这是一个包。 |
| **deps.py** | 放「大家共用的依赖」（比如以后做登录校验、访问频率限制）。目前是占位，方便以后加功能。 |
| **routes.py** | 定义两个对外的「接口」：一个是「上传音频，返回文字」，一个是「传一段文字+目标语言，返回翻译」。里面会检查文件类型、大小、参数是否合法，然后调用下面的服务，把结果返回。 |

### backend/app/schemas/（请求和响应的数据格式）

| 文件 | 通俗说明 |
|------|----------|
| **__init__.py** | 标记包，并统一导出下面定义的数据格式，给路由用。 |
| **transcribe.py** | 定义「转写接口」返回的数据长什么样（比如就一个字段：转出来的文字）。这样前后端约定一致，也方便做校验。 |
| **translate.py** | 定义「翻译接口」：前端要传什么（要翻的文字、目标语言），后端返回什么（翻译后的文字）。同样是为了格式统一和校验。 |

### backend/app/services/（真正调外部服务的地方）

| 文件 | 通俗说明 |
|------|----------|
| **__init__.py** | 标记包。 |
| **transcribe.py** | 「转写服务员」。收到音频数据后，调用外部的语音转文字服务（目前是 OpenAI Whisper），把结果文字拿回来。密钥从 config 里读，不写死在代码里。 |
| **translate.py** | 「翻译服务员」。收到一段文字和目标语言后，调用外部的翻译服务（目前是 OpenAI），把翻译结果拿回来。同样从 config 读密钥，并且预留了以后换成别的翻译服务（比如阿里云）的扩展方式。 |

## frontend/（前端：你在浏览器里看到的界面）

前端负责：让你选择音频、点击转写/翻译、把结果展示出来，并提供复制、下载。**前端只和后端自己的接口说话，不接触任何密钥。**

### 前端根目录

| 文件 | 通俗说明 |
|------|----------|
| **.dockerignore** | 做前端「打包成镜像」时，不把本地依赖、构建缓存等打进去，让镜像更小、更干净。 |
| **Dockerfile** | 前端的「打包说明书」：先在一个阶段里安装依赖、打包出静态网页；再在另一个阶段里用一个小型网页服务器（nginx）把这些页面和资源对外提供，并把对「后端接口」的请求转给后端容器。 |
| **nginx.conf** | 给 nginx 用的「路由与转发规则」：哪些地址是页面、哪些地址要转发到后端。这样浏览器访问同一个域名就能既打开页面又调后端，不用操心跨域。 |
| **index.html** | 整个前端单页的「外壳」：一个最简的 HTML，里面有一个挂载点，真正的界面由 React 填进去。 |
| **package.json** | 前端的「依赖与脚本清单」：用到哪些库（React、构建工具等）、以及「开发时运行」「打包」「预览」等命令。运行 `npm install` 和 `npm run dev` 都靠它。 |
| **tsconfig.json** | TypeScript 的「编译与检查规则」：用哪种语法、多严格等，让整个前端代码风格统一、少出错。 |
| **tsconfig.node.json** | 给构建工具自身用的 TypeScript 配置（例如 vite 的配置文件），和页面代码的配置分开。 |
| **vite.config.ts** | 前端构建/开发工具的配置：用哪个端口、怎么把对「后端接口」的请求在开发时转发到本机后端，这样你改前端代码时不用改地址。 |

### frontend/src/（前端源代码）

| 文件 | 通俗说明 |
|------|----------|
| **main.tsx** | 前端的「入口」：把 React 装进页面里、挂上根组件、引入全局样式。打开页面时最先执行的就是这里。 |
| **App.tsx** | 整个页面的「总控」：负责「选文件 → 转写 → 选语言 → 翻译 → 展示/复制/下载」这一条流程，以及调用后端的时机、报错提示等。展示文字时只用安全方式，避免恶意脚本。 |
| **index.css** | 全站共用的样式：背景、字体、按钮、输入框、错误提示的大致样子，让界面统一好认。 |
| **vite-env.d.ts** | 给 TypeScript 用的类型说明，主要是「构建工具提供的环境变量」长什么样（比如前端配置的后端地址），避免写错名字或类型。 |

### frontend/src/api/

| 文件 | 通俗说明 |
|------|----------|
| **client.ts** | 前端的「和后端对话员」。封装好「上传音频做转写」「发文字做翻译」两次请求，页面里只要调这两个方法，不用自己写地址、拼参数。后端地址可以配置，这里不写任何密钥。 |

### frontend/src/components/（页面上的小块界面）

| 文件 | 通俗说明 |
|------|----------|
| **FileUpload.tsx** | 「选文件」那一块：让你点选一个音频文件，选好后交给上层去触发转写。可能还带一个「正在转写」的提示。 |
| **TranscribeResult.tsx** | 「转写结果」那一块：把转出来的文字显示在一个框里，并有一个「翻译」按钮，点完就去请求翻译。 |
| **TranslatePanel.tsx** | 「翻译」那一块：选择要翻译成哪种语言、显示翻译结果、提供「复制到剪贴板」和「下载成 txt 文件」的按钮。 |

### frontend/src/types/

| 文件 | 通俗说明 |
|------|----------|
| **api.ts** | 定义前端和后端之间「传的数据长什么样」：转写返回什么、翻译要传什么、返回什么。这样写代码时有自动提示、类型检查，前后端约定也更清楚。 |

## 小结

- **根目录**：告诉 Git 什么别上传、告诉人怎么用、告诉 Docker 怎么一起起前后端。  
- **backend**：读配置、接请求、调语音转写和翻译、把结果返回；密钥和敏感配置只在这里。  
- **frontend**：你看到的页面、选文件、点按钮、看结果、复制下载；只和后端自己的接口通信，不碰密钥。  

这样分工后，既能本地用，也能用 Docker 一键跑，以后要部署到云上或嵌入别的系统也容易说清楚「哪块是干啥的」。
